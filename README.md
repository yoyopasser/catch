# 台股決策工具

把「資料品質 → 趨勢結構 → 均線方向 → 所處位置 → 支撐壓力 → 進場型態 → 收盤確認 → 停損與部位 → 持有／減碼／退出」改寫成固定、可重複執行的網頁決策流程。

工具不保證明日漲跌，也不會為了每天提供答案而勉強推薦股票；「等待」、「資料不足」與「今日沒有合格標的」都是正式結論。

## 在 GitHub Codespaces 一鍵開啟

[![Open in GitHub Codespaces](https://github.com/codespaces/badge.svg)](https://codespaces.new/yoyopasser/catch?quickstart=1)

1. 按上方按鈕並登入 GitHub。
2. 選擇 **Create codespace**。
3. 第一次會自動安裝套件並執行測試。
4. 伺服器會自動在 `8501` 連接埠啟動，瀏覽器通常會自動開啟網站。
5. 若沒有自動開啟，在 Codespaces 下方的 **Ports** 頁籤，點選 `8501` 的網址。

### 重要隱私設定

- `8501` 連接埠請維持 **Private**，不要改成 Public；目前網站沒有帳號登入功能。
- Repository 不包含任何交易資料、行情快取或 Token。
- 個人資料保存在 Codespace 的 `data/stock_tool.db`，不會被 Git 提交。
- Codespace 被刪除後，本機資料也會消失。請定期到「設定與資料」下載完整備份。

## Codespaces 日常使用

Codespace 停止後，再從 GitHub repository 點選 **Code → Codespaces** 開啟原本的 Codespace，網站會自動重新啟動。若要手動重啟：

```bash
bash .devcontainer/start-server.sh
```

查看伺服器紀錄：

```bash
cat .codespace/server.log
```

## 第一次操作

1. 進入「設定與資料」，按 **更新上市櫃清單**。
2. 進入「全市場篩選」，先選「成交值前200檔」確認資料源正常。
3. 從候選名單查看完整理由，將真正準備追蹤的股票加入鎖定清單。
4. 每日盤後按「更新並重新分析」。
5. 有買賣時，立即在「交易紀錄」登記。
6. 系統會依持有成本、停損、MA5、MA20、前低及結構狀態調整隔日方案。

## 資料備份與還原

「設定與資料」頁面提供：

- **下載完整備份**：輸出風險設定、自選股與交易紀錄。
- **還原並取代目前資料**：將備份匯入新的或既有 Codespace。

備份刻意排除 FinMind Token；Token 請在新環境重新輸入。還原前應先更新上市櫃清單。

## 資料機制

### 最新盤後資料

- 上市：TWSE OpenAPI `STOCK_DAY_ALL`。
- 上櫃：TPEx OpenAPI 每日收盤行情。

### 歷史K線

預設使用 Yahoo Chart 介面逐檔取得並快取於 `data/cache/history`。免費資料可能延遲、缺漏或受服務限制，並非交易所授權的專業即時行情。

### 盤中資料

鎖定清單可透過 TWSE MIS 取得最新可用報價，只作盤中暫估；盤中刺穿壓力不會被當成正式突破。

### 為何仍需要歷史資料

即使每次開啟都抓最新價格，以下條件仍需要歷史K線：

- 頭頭高、底底高／頭頭低、底底低。
- MA5、MA20、MA60及方向。
- ATR、量能基準、前高前低與盤整頸線。
- 突破是否發生在合理位置。
- 停損、下一壓力與風險報酬比。

因此系統採用「最新行情接到歷史序列後重新運算」，不是只靠歷史資料猜測今天，也不是只看當下價格。

## 全市場掃描模式

- 前200檔：快速確認。
- 前500檔：日常建議。
- 前1000檔：擴大範圍。
- 全部上市櫃：最完整，但首次會大量抓取歷史資料，較容易受到免費資料源限制。

排名第一層使用最新成交值，先排除多數流動性過低的股票。

## 決策規則摘要

### 多頭資格

- 最近兩個確認高點提高。
- 最近兩個確認低點提高。
- 股價最好位於上揚MA20與MA60之上。

### 進場候選

- 盤整上緣由收盤價正式突破。
- 多頭正常回檔後，以紅K突破前一日高點。
- ABC下降修正線被收盤突破。
- 量能、K棒實體、所處位置及風險報酬比共同評分。

### 不追價

股價距MA20過遠時，以ATR衡量並扣分；跳空使停損距離過大時，隔日方案會要求等待回測。

### 持有部位優先規則

1. 跌破停損或確認前低：全部退出。
2. 跌破MA20：減碼並提高警戒。
3. 已有獲利且高檔放量長黑：優先停利。
4. 跌破MA5：可分批減碼。
5. 未觸發上位條件：續抱。

### ETF差異

- 一般股票ETF可使用趨勢、均線、前高低與區間突破。
- ETF的K棒幅度與量能門檻比個股低。
- 海外ETF提醒原型指數、匯率與折溢價。
- 債券ETF提醒利率與存續期間。
- 商品ETF提醒期貨轉倉。
- 槓桿／反向ETF只列為短線工具並降低分數。

## 分數的意義

分數只用於候選排序，不是上漲機率：

- 70以上：條件較完整，列為隔日優先候選。
- 55～69：觀察或依風險額度小部位試單。
- 低於55：通常等待、結構未明或風報比不足。

停損、資料失效與結構破壞永遠優先於分數。

## 本機執行

### Windows

1. 安裝 Python 3.11以上版本。
2. 下載並解壓縮專案。
3. 雙擊 `啟動工具.bat`。
4. 開啟 `http://127.0.0.1:8501`。

### macOS／Linux

```bash
./啟動工具.sh
```

## 測試

```bash
pip install -r requirements-dev.txt
python -m pytest -q
```

GitHub Actions 也會在每次推送時自動執行測試。

## 重要限制

- 技術分析只能建立條件與風險控制，不能客觀保證明日方向。
- 免費行情可能延遲、缺漏或被限制。
- 公司重大訊息、處置股、交易停止、財報與流動性風險仍需另行確認。
- 系統目前以做多與持有管理為主，未把融券、借券成本與做空資格納入下單建議。
- 本工具是研究與紀律輔助，不構成個別投資建議。
